# v1 MVP — council binary implementation

## Overview

Build the first working council binary per the v1 design docs. Single-pass multi-expert CLI committee: one question → N expert claude-code subprocesses in parallel → quorum gate → judge synthesizes one final answer → artifacts on disk under `./.council/sessions/<id>/`.

MVP ships a two-persona default profile (`independent` + `critic`) served by the same `claude -p` CLI — "single CLI, multiple personas" per ADR-0005. Forward-compat shapes for v2 debate are baked in without activating them (session folder nested under `rounds/1/`, `verdict.json.rounds[]` array, `pkg/prompt` accepts optional `prior_rounds` parameter, pluggable `Executor` interface for future Codex / Gemini CLIs).

Integrates with existing design:
- `docs/design/v1.md` §3 Architecture defines the pipeline shape
- `docs/design/v1.md` §5 Config defines `default.yaml` schema
- `docs/design/v1.md` §6 Session folder defines on-disk layout (rounds-nested)
- `docs/design/v1.md` §7 Executor interface is the CLI plugin contract
- `docs/design/v1.md` §8 Judge role names the four jobs (synthesis / format / injection boundary / canonical artifact)
- `docs/design/v1.md` §10 Error handling defines timeouts, 429 distinction, SIGINT behavior
- `docs/design/v1.md` §16 Fitness functions F1–F7 are the acceptance criteria
- ADRs 0001–0006 carry load-bearing decisions

### Task 3: Scaffold `pkg/runner` subprocess primitive

- [ ] create `pkg/runner/runner.go` — `RunRequest` (fields: `Argv []string`, `Prompt string`, `Env []string`, `StdoutFile, StderrFile string`, `Timeout time.Duration`, `MaxRetries int`) and `RunResponse` (fields: `ExitCode int`, `Duration time.Duration`, `Retries int`, `RateLimited bool`). `Env` gets passed to `cmd.Env` so callers (e.g. `pkg/executor/claudecode`) can set `CLAUDE_CODE_MAX_OUTPUT_TOKENS=64000` without reaching into `pkg/runner` internals.
- [ ] `Run(ctx context.Context, req RunRequest) (RunResponse, error)` — spawn with `exec.Cmd`, set `SysProcAttr{Setpgid: true}` and `cmd.Env = req.Env` (if non-nil), pipe `req.Prompt` to stdin; stdout → `req.StdoutFile`; stderr → `req.StderrFile`
- [ ] timeout via `context.WithTimeout(ctx, req.Timeout)`; on fire: `pgid, _ := syscall.Getpgid(cmd.Process.Pid); syscall.Kill(-pgid, syscall.SIGTERM)`, 2s grace, then `syscall.Kill(-pgid, syscall.SIGKILL)`. Using `Getpgid` (not bare `-cmd.Process.Pid`) matters: if the child has called its own `setpgid`, the bare form targets the wrong group.
- [ ] define sentinel errors: `ErrTimeout`, `ErrNonZeroExit`, `ErrRateLimit`
- [ ] create `pkg/runner/ratelimit.go` — after non-zero exit, `scanStderr(path string) (retryAfter time.Duration, ok bool)` regex-matching `(?i)(rate[_ ]limit(?: exceeded)?|\b429\b.*too many|too many.*\b429\b)` and attempting to extract a `Retry-After: <N>s` hint via `Retry-After:\s*(\d+)s?`. Narrowed from a bare `429` to `\b429\b` with a co-occurrence marker to avoid false matches on port numbers or byte counts in stderr.
- [ ] retry loop in `Run`: on `ErrRateLimit` sleep retry-after-hint-or-10s, retry up to `req.MaxRetries + 1` times (rate-limited attempts don't consume a retry slot on the caller side — they do here). On `ErrTimeout` or `ErrNonZeroExit` without rate-limit marker: retry up to `req.MaxRetries` times.
- [ ] **Retry-ownership comment at top of `runner.go`:** rate-limit retries are runner-owned (infrastructure concern — all callers inherit the same throttling behavior). Fail-retries (timeout, non-zero exit) are orchestrator-owned (policy concern — per-profile `max_retries`); callers pass `MaxRetries: 0` to `Run` to disable runner-side fail-retry. This split keeps infra vs policy separable.
- [ ] delete `StderrFile` on successful exit (exit 0) to keep session folder clean per design/v1.md §7
- [ ] write tests with stub commands: `sh -c "exit 0"` → success path; `sh -c "sleep 5"` + 1s timeout → `ErrTimeout`, process tree dead (check with `pgrep` in test); `sh -c "echo rate_limit >&2; exit 1"` → `ErrRateLimit` + retry count == 1 + eventual failure (with `MaxRetries=0`)
- [ ] write tests for the process-group kill: parent spawns a script that `sh -c "sleep 30 & sleep 30"` and the test verifies no orphan `sleep` remains after `Run` times out
- [ ] run `go test ./pkg/runner/...` — must pass before Task 4

### Task 6: `pkg/orchestrator` fan-out + quorum + judge + verdict assembly

- [ ] create `pkg/orchestrator/orchestrator.go` — `Run(ctx context.Context, profile *config.Profile, question string, sess *session.Session) (*session.VerdictV1, error)`
- [ ] fan-out phase: one goroutine per expert; each goroutine looks up `Executor` via `executor.Get(profile.Experts[i].Executor)`, builds prompt via `pkg/prompt.BuildExpert`, writes expert prompt to `<expertdir>/prompt.md`, calls `Executor.Execute` with stdout → `<expertdir>/output.md` and stderr → `<expertdir>/stderr.log`
- [ ] per-expert retry policy: orchestrator-level retry up to `profile.MaxRetries` times on `ErrTimeout`/`ErrNonZeroExit` (but NOT on `ErrRateLimit` — runner already handles rate-limit waits + retries). Record retry count into `verdict.Experts[i].Retries`.
- [ ] on expert success: delete `stderr.log` (design §7: persisted only on failure), `sess.TouchDone(expertDir)`. On failure after retries: record `status: "failed"` and `exit_code` and `duration_seconds` in verdict; leave `stderr.log` in place (persisted on failure); do NOT write `.done`.
- [ ] both expert and judge goroutines invoke the same `Executor.Execute` call path (no per-role duplication) — this satisfies architect-review P2 "Subprocess primitive duplicated between expert and judge". Add a test asserting `pkg/runner.Run` is called with the same code path for both roles (e.g., via a shared interface or a grep assertion in the orchestrator package).
- [ ] create `pkg/orchestrator/quorum.go` — after all expert goroutines complete: `if count(succeeded) < profile.Quorum: sess.WriteVerdict(v with status="quorum_failed"); return exit-code-2-signal`
- [ ] judge phase: `pkg/prompt.BuildJudge` with the surviving experts, write judge prompt to `<judgedir>/prompt.md`, `Executor.Execute` with stdout → `<judgedir>/synthesis.md`, stderr → `<judgedir>/stderr.log`. Judge retry once on failure; if both attempts fail → exit-code-3-signal with `status: "judge_failed"`.
- [ ] on judge success: delete `stderr.log`, `sess.TouchDone(judgeDir)`, set `verdict.Answer = contents of synthesis.md`, `verdict.Status = "ok"`. On judge failure after retry: `stderr.log` persisted, no `.done` written, orchestrator returns exit-3 sentinel.
- [ ] signal handling: the caller (cmd/council) owns SIGINT; orchestrator respects `ctx.Done()` — when context cancelled, goroutines cleanup (their kill is already handled by `pkg/runner`), orchestrator writes `verdict` with `status: "interrupted"` listing which experts had already completed (`status: "ok"` for completed, `status: "interrupted"` for in-flight, absent for never-started), returns a sentinel `ErrInterrupted` so main can exit 130.
- [ ] atomic verdict write is the LAST thing the orchestrator does, via `sess.WriteVerdict` — including on the interrupted path (verdict must be on disk BEFORE the orchestrator returns, so main can exit 130 with the verdict visible).
- [ ] write tests with a stub executor (table-driven via `executor.Register` of a test impl): success path (all experts ok → verdict `status: "ok"`); F4 case (stub fails once-then-ok → `verdict.rounds[0].experts[].retries == 1`); quorum failure (2 of 2 fail with quorum=1 → status `quorum_failed`); judge failure (experts ok, judge always fails → `judge_failed`, exit 3); SIGINT partial (expert 1 completes, SIGINT fires before expert 2 finishes → `verdict.rounds[0].experts[0].status == "ok"`, `experts[1].status == "interrupted"`, judge section absent, top-level `status == "interrupted"`); stderr cleanup on success (expert `stderr.log` file does not exist after `status: "ok"`).
- [ ] run `go test ./pkg/orchestrator/...` — must pass before Task 7

## Technical Details

- **verdict.json schema v1**: top-level `version: 1`, `session_id`, `session_path`, `profile`, `question`, `answer`, `status`, `started_at`, `ended_at`, `duration_seconds`; `rounds: [{experts: [{name, executor, model, status, exit_code, retries, duration_seconds}], judge: {executor, model, exit_code, retries, duration_seconds}}]`. Version stays 1 for all of v1; v2 bumps to 2.
- **Session ID**: `<YYYY-MM-DDTHH-MM-SSZ>-<adj>-<adj>-<noun>` (e.g. `2026-04-19T17-02-14Z-fizzy-jingling-quokka`). Petname via `golang-petname` with separator `-` and words count 3.
- **Process-group kill**: `pgid, _ := syscall.Getpgid(cmd.Process.Pid); syscall.Kill(-pgid, syscall.SIGTERM)` then after 2s `syscall.Kill(-pgid, syscall.SIGKILL)`. Requires `SysProcAttr{Setpgid: true}` at spawn. Using `Getpgid` (not bare `-cmd.Process.Pid`) handles the case where the child calls its own `setpgid`.
- **429 regex**: `(?i)(rate[_ ]limit(?: exceeded)?|\b429\b.*too many|too many.*\b429\b)`. `Retry-After: <N>s` hint extracted with `Retry-After:\s*(\d+)s?`. Bare `429` was rejected to avoid false matches on port numbers / byte counts.
- **Retry ownership**: rate-limit retries are in `pkg/runner` (infrastructure); fail-retries (timeout, non-zero exit) are in `pkg/orchestrator` (policy, uses `profile.max_retries`). `pkg/executor/claudecode` calls `pkg/runner.Run` with `MaxRetries: 0` to disable runner-side fail-retry.
- **Quorum**: survivors_count `<` `profile.Quorum` → fail. Note strict less-than; `survivors == quorum` passes.
- **Atomic rename**: `O_CREATE|O_WRONLY|O_EXCL` open for `.tmp`, write, `f.Sync()`, `f.Close()`, `os.Rename`. Readers see either no file, the old file, or the new file — never partial.
